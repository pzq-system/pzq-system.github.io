    # 生成SSH Key 密钥

    # 1，打开Git Bash 命令行窗口

    如果没有会返回如下信息

    $ cd ~/.ssh
bash: cd: /c/Users/Him/.ssh: No such file or directory
    1
    2

    已经有创建SSH，会返回如下信息，表示本地有SSH了

    $ cd ~/.ssh
$ ls
id_rsa  id_rsa.pub  known_hosts

    2
    3

    # 2，创建SSH Key

    $ ssh-keygen -t rsa -C "GitHub注册的邮箱"
    1

    输入生成的key存放的路径，不设置 会默认 创建在C:/Users/用户账号/.ssh文件夹

    # Git

    下载地址 https://git-scm.com/download/win

    # 本地库初始化

    git init

    # 设置Git签名

    项目级别签名，设置指令

    git config user.name 用户名
git config user.email 邮箱
    1
    2

    对于系统级别签名，设置指令有些许不同

    git config --global user.name 用户名
git config --global user.email 邮箱
    1
    2

    # 添加、查看状态、提交操作

    查看状态指令

    git status
    1
    • On branch master：表示我们目前处在master分支上，关于分支的概念后面会详细讲解
    • No commits yet：没有任何的提交，表示本地库中没有任何的提交
    • nothing to commit：没有什么可提交，表示暂存区没有什么可提交的东西

    添加指令

    git add .
    1

    我们通过git add 指令追踪到文件，也就是将文件放入了暂存区，这时候，我要想反悔，我就可以通过该指令将刚才放入暂存区的文件撤回来，执行指令：

    git rm --cached 撤回的文件
    1

    提交

    git commit -m '备注'
    1

    # 查看历史记录三种方式

    git log
    1
    git log --pretty=oneline
    1
    git log --oneline
    1

    git reflog能够显示所有版本的提交历史

    git reflog
    1

    这种方式显示提交历史的区别在于，它多了一个信息：HEAD@{0}。

    切换版本

    切换版本时哈希值不必全部写出来，只需要写出部分能够唯一标识当前版本即可。

    git reset --hard 14309ac
    1

    找回被删除的文件

    1. 删除文件已经提交到了本地库：此时使用git reset --hard [版本索引值]指令回退到删除文件前的版本即可
    2. 删除文件添加到了暂存区，但还未进行提交：此时使用git checkout -- 文件名刷新一下三大区即可
